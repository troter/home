# コミットメッセージレビュー

既存のコミットメッセージをコミットルールに照らして評価し、改善案を提案する。

## 手順

1. 引数で指定された範囲のコミットログを `git log --no-merges --format="commit %H%n%B---END-COMMIT---" <範囲>` で取得する（デフォルト: 直近1コミット = `HEAD~1..HEAD`）
2. コミットルール（`~/.claude/rules/commit.md`）に照らして各コミットメッセージを評価する。ルールがコンテキストにない場合は先にファイルを読み込む
3. 問題点と改善案を提示する

## 評価観点

- subject が日本語・命令形（「〜する」）になっているか
- type が適切か
- subject が変更内容を正しく要約しているか（問題の記述ではなく修正内容の要約か）
- body が必要な場合に書かれているか（問題・解決・代替案の構造）
- 1コミット1目的の原則が守られているか — `git show --stat <hash>` で変更範囲を確認し、無関係なファイルが混在していないか判断する
- Conventional Commits のフォーマットに従っているか

## 出力フォーマット

各コミットについて以下を出力する:

- コミットハッシュと現在のメッセージ
- 評価（良い点・問題点）
- 改善案（問題がある場合、修正後のメッセージをコードブロックで提示）
- 分割が必要な場合は、推奨する分割単位と各コミットのメッセージ案を提示する

## 引数の扱い

- 引数がある場合: `git log` の範囲指定として使う（例: `HEAD~3..HEAD`, `main..feature`）
- 引数がない場合: 直近1コミット（`HEAD~1..HEAD`）をレビューする
